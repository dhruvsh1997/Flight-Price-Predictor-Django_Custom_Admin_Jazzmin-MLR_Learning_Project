## 5. Multiple Linear Regression — Deep Dive

### What is MLR?

Multiple Linear Regression models the relationship between **one continuous target** (price) and **multiple predictor variables** (features):

```
price = β₀ + β₁·x₁ + β₂·x₂ + … + βₙ·xₙ + ε
```

Where:
- `β₀` = intercept (base price when all features are 0)
- `β₁…βₙ` = coefficients (how much price changes per unit of each feature)
- `ε` = error term (residual)

### Feature Pipeline (what `test.py` builds)

```
Raw DataFrame
    ↓
MultiColumnLabelEncoder   ← converts strings to integers per column
    ↓                        e.g. "Vistara" → 5, "Delhi" → 1
StandardScaler            ← zero-mean, unit-variance scaling
    ↓                        prevents large-valued features dominating
LinearRegression          ← fits β₀ … βₙ via Ordinary Least Squares
    ↓
Predicted Price (INR)
```

The **entire pipeline** is saved as a single `.pkl` file so the Django app loads it once and runs `pipeline.predict()` — no re-encoding needed.

### Encoding Strategy

| Approach | Used here | Why |
|----------|-----------|-----|
| `LabelEncoder` | ✅ | Assigns stable integer codes; classes_ list lets us audit mappings |
| `OrdinalEncoder` | ❌ | Wraps LabelEncoder, less transparent for learning |
| `OneHotEncoder` | ❌ | Expands dimensions; MLR works fine with ordinal encoding on small cardinality |

> **Learning note:** `LabelEncoder` implies ordinal ordering (0 < 1 < 2…) which isn't always valid for nominal categories like airline names. For production, OneHotEncoder is more correct. This project uses `LabelEncoder` deliberately so you can see the integer mappings and understand coefficient interpretation.

### Evaluation Matrices (printed by `test.py`)

| Metric | Formula | Interpretation |
|--------|---------|----------------|
| **MAE** | `mean(|y - ŷ|)` | Average absolute error in rupees — easy to interpret |
| **MSE** | `mean((y - ŷ)²)` | Penalises large errors more heavily |
| **RMSE** | `√MSE` | Same units as price (INR); most commonly reported |
| **R²** | `1 - SS_res/SS_tot` | Proportion of variance explained (0–1; higher = better) |
| **Adj. R²** | `1 - (1-R²)·(n-1)/(n-p-1)` | R² adjusted for number of predictors; use this when comparing models |
| **5-fold CV R²** | Cross-validated R² | More robust than single train/test split |

### Coefficient Interpretation

After training, `test.py` prints each feature's coefficient:

```
Coefficient airline          : +1234.56
Coefficient source_city      :  -456.78
Coefficient class            : +8901.23
Coefficient duration         : +2100.00
Coefficient days_left        :  -150.00
```

- **Positive coefficient** → feature value ↑ = price ↑
- **Negative coefficient** → feature value ↑ = price ↓
- `days_left` typically has a **negative coefficient** — booking earlier → cheaper fares
- `class` has a large **positive coefficient** — Business class costs much more

### MLR Assumptions (for learning)

1. **Linearity** — relationship between features and price is linear
2. **No multicollinearity** — features aren't highly correlated with each other
3. **Homoscedasticity** — error variance is constant across all predictions
4. **Normality of residuals** — prediction errors are normally distributed

> Flight prices violate some of these (especially linearity for Business vs Economy), which is why the R² may not reach 1.0. This is a great conversation point when learning!

```
                                            │ session_key  │
                                            └──────────────┘
```

### Why normalise?

| Without normalisation | With normalisation |
|----------------------|-------------------|
| "Vistara" stored 300K× | "Vistara" stored once in Airline table |
| Typo in one row corrupts reports | Single source of truth |
| `GROUP BY airline` on strings = slow | `GROUP BY airline_id` on int = fast |
| Changing "GO_FIRST" → "Go First" = full scan | Update one row in Airline |

## 8. Logging Architecture

All logging is configured in `settings.py` under `LOGGING`.

### Two destinations
| Handler | File | Level | Rotation |
|---------|------|-------|---------|
| `console` | stdout | DEBUG | — |
| `file_django` | `logs/django.log` | INFO | 5 MB × 3 files |
| `file_predictor` | `logs/predictor.log` | DEBUG | 5 MB × 3 files |

### Three loggers
| Logger name | Used in | Level |
|------------|---------|-------|
| `django` | Django internals | INFO |
| `predictor` | All app views, models, ml_engine | DEBUG |
| `ml_training` | `test.py` only | DEBUG |

### Log format
```
[2024-03-15 14:32:01] [INFO] [predictor:views:58] — Prediction request: Vistara | Delhi → Mumbai | Business | dur=2.5 | days=14
```

## 10. How `test.py` Works

```
python test.py
```

**Step-by-step:**

| Step | What happens |
|------|-------------|
| 1 | Loads `dataset/Clean_Dataset.xlsx` with pandas |
| 2 | Normalises column names, drops unused columns (`flight`), handles NaN |
| 3 | Seeds all 6 DB lookup tables + bulk-inserts FlightRecord rows |
| 4 | Builds feature matrix X (9 features) and target y (price) |
| 5 | 80/20 train-test split with `random_state=42` |
| 6 | Builds Pipeline: `MultiColumnLabelEncoder → StandardScaler → LinearRegression` |
| 7 | Evaluates: MAE, MSE, RMSE, R², Adj. R², 5-fold CV, sample predictions |
| 8 | Saves pipeline to `ml_models/flight_price_model.pkl` with `joblib.dump` |

**Re-running `test.py`:** The DB seeding step skips FlightRecord insertion if records already exist. Delete `db.sqlite3` to start fresh.

## 13. Common Errors & Fixes

| Error | Cause | Fix |
|-------|-------|-----|
| `FileNotFoundError: dataset/Clean_Dataset.xlsx` | Dataset not placed | Download from Kaggle, put in `dataset/` |
| `FileNotFoundError: ml_models/flight_price_model.pkl` | Model not trained | Run `python test.py` |
| `No module named 'jazzmin'` | Dependencies not installed | `pip install -r requirements.txt` |
| `no such table: predictor_flightrecord` | Migrations not run | `python manage.py migrate` |
| Form dropdowns empty | Lookup tables not seeded | Run `python manage.py seed_lookups` or `python test.py` |
| `KeyError: 'AirAsia'` in ml_engine | Airline name mismatch | Check `AIRLINE_MAP` in `ml_engine.py` matches training data |
